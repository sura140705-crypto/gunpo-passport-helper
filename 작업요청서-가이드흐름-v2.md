# 여권발급신청서 도우미 — 가이드 흐름 개선 작업 요청 (v2, 최종)

## 0. 대전제 (절대 준수)
- **좌측 서식 표기(`CO` 맵 line ~537, `renderForm()` line ~575)는 수정 금지.** 완성·검증된 영역이다.
- **🔴 데이터 계약(가장 중요)**: 선택지 필드가 `state.data`에 **저장하는 값**은 `CO.checks[field]`의 **키 문자열과 글자까지 정확히 일치**해야 한다. 좌측 폼은 `CO.checks[field][ state.data[field] ]`로 체크 위치를 찾으므로, 값이 안 맞으면 `undefined`가 되어 **✔가 안 찍힌다.** 버튼 라벨은 자유롭게 바꾸되 저장값은 아래 표를 지킬 것(1항 참조). → 좌측을 안 건드려도 데이터 계약이 깨질 수 있으니 최우선 확인.
- **단일 HTML, 외부 리소스 0.** CDN·폰트·라이브러리·서버·API 호출 추가 금지.
- **저장 0.** localStorage 등 금지. `state`는 메모리 전용.
- **개인정보**: 이름·주민번호·주소는 브라우저 메모리에서만 처리, 전송·저장 금지.
- **어르신 친화** 톤 유지(큰 글씨·쉬운 말·"잘 모르겠어요" 배려).
- **재사용성**: 상단 `ORG_CONFIG`만 바꿔 타 민원실 재사용 가능하게, 하드코딩 지양.

## 1. 🔴 선택지 저장값 계약 표 (필수 준수)
버튼에 친화적 라벨(예: "우편 등기로 받기")을 쓰더라도, `state.data[field]`에는 아래 정확한 값을 저장한다. (스키마 `store` 키로 라벨↔저장값을 분리 — 5항 참조)

| field | 반드시 저장할 정확한 값 | 비고 |
|---|---|---|
| `kind` | `일반` `관용` `외교관` `긴급` `여행증명서` | |
| `travelType` | `왕복` `편도` | `kind==="여행증명서"`일 때만 |
| `pages` | `26면` `58면` | |
| `period` | `10년` `단수(1년)` `잔여기간` `5년` `5년 미만` | |
| `braille` | `희망` `희망 안 함` | 시각장애인만 |
| `mailService` | `희망` `희망 안 함` | **"우편 등기 수령"→`희망`, "방문 수령"→`희망 안 함"** |

- `hasPassport`는 **서식에 칸이 없는 로직 전용 필드**(로마자 분기 스위치). `CO`에 없으니 값 형식 자유(예: `있음`/`없음`)로 두면 됨.
- `regBase`/`mailAddr`는 자유 텍스트라 매핑 불필요(완성 문자열을 그대로 저장).

## 2. 작업 목표
가이드 흐름(`STEP_DEFS` + `renderStep` + 유틸)을 **더 단순하고 편리하게** 재구성한다.
핵심 원칙: **스마트 기본값 + 점진적 공개(대부분 스킵, 필요 시 펼침)**.

## 3. 흐름 재설계 (11단계 → 7단계 + 준비물)

| 단계 | 내용 | 핵심 처리 |
|---|---|---|
| 0 | **준비물 확인** | 신분증·사진 체크리스트. **차단 아님(확인만)**, 없어도 진행 가능·"직원 문의" 안내 |
| 1 | **여권 종류·면수·기간 통합** | 한 화면에서 **기본값 미리 선택**: 일반여권·10년·58면. "표시 없으면 이대로 발급된다"는 서식 근거를 쉬운 말로 설명 |
| 2 | **한글성명 + 주민번호 + 본인연락처** | 신청인 기본정보 |
| 3 | **기존 여권 있음/없음** | 이 선택이 4단계 로마자의 **분기 스위치**(`hasPassport`) |
| 4 | **로마자 표기** | 없음(신규)→직접입력(권장 예시 제안) / 있음→**"여권 뒷장과 글자 하나까지 똑같이" 경고 강조**, 변경 시에만 수정 |
| 5 | **긴급연락처** | 성명·관계·전화 한 화면. **본인 번호 입력 시 경고**("해외 사고 대비, 다른 분 번호") |
| 6 | **우편 수령 여부** | ★ 독립 단계로 분리(5항 상세) |
| 7 | **요약 / 미리보기** | (a) 적은 내용 (b) **직원 확인 항목**(불확실·미입력) 2분할 |

- **등록기준지·배우자 로마자·점자** 등 추가기재는 **기본 스킵**. "대부분 안 적으셔도 됩니다" 게이트(`gate`)로 묶고, **담당자 보완 요청 시** 접이식으로 펼쳐 입력.
- **우편 수령만은 게이트에서 제외**하고 독립 단계로 둔다(수요 있음 + 요금·주소 안내 필요).

## 4. 스마트 기본값 (단계 1)
- 종류=일반여권 / 기간=10년 / 면수=58면을 **미리 선택된 상태**로.
  → `createEmptyState()`의 `data`에 `kind:"일반", period:"10년", pages:"58면"`을 기본 세팅. (저장값은 1항 표 준수)
  → ⚠️ 이 경우 **좌측 서식에 처음부터 ✔가 찍혀 나옴**(의도된 동작). **초기화 버튼도 이 기본값으로 리셋**되게 할 것.
- 각 기본값 옆 이유 한 줄: 예) "가장 많이 신청하세요 · 표시 안 하면 이대로 발급됩니다."
- 대부분 그대로 [다음] 누르면 되게.

## 5. 우편 수령 단계 (신규·핵심)
주소는 **오프라인 구조화 입력**으로 처리(자동검색 API 미도입).

- `mailService` 선택(저장값은 1항 표): **"방문 수령"(기본·추가요금 없음) → `희망 안 함`** / **"우편 등기 수령"(요금 별도) → `희망`**.
- '우편 등기' 선택 시에만 펼침:
  - **요금 안내 배너**: "등기요금 별도, 정확한 금액·도착일은 직원 확인" — **금액 하드코딩 금지**(요금표 변동).
  - **구조화 주소 입력** `type:"addr"` 신규 도입:
    - 시/도(드롭다운, `SIDO` 상수) / 시·군·구 / 도로명 / 건물번호 / 상세주소(동·호)
    - 입력 시 `composeAddr()`로 **도로명 어순 자동 조합** → `state.data.mailAddr`에 저장
    - "서식에 이렇게 들어갑니다 →" 실시간 미리보기 표시
- `state.data.mailAddr`는 완성 문자열이므로 **좌측 `CO.texts.mailAddr`가 `renderForm` 수정 없이 자동 반영**됨(수정 금지 영역 미접촉 ✅). `mailAddr`는 이미 wrap 설정(1줄 중앙/길면 2줄)돼 있음.

## 6. 스키마 확장 (선 합의 후 진행)
개별 단계 작업 전에 `STEP_DEFS` 필드 스키마에 아래 키를 먼저 정의한다.
- **`store`** — 선택지 라벨↔저장값 분리. opt에 `{ label:"우편 등기로 받기", value:"희망" }` 형태(또는 필드에 `store` 매핑). **1항 계약을 지키는 핵심 키.**
- `example` — 로마자 등 권장 예시(확정 아님).
- `warn: { when:d=>bool, msg }` — 조건부 경고(예: 기존여권 있음 → 로마자 일치 경고 / 긴급연락처=본인번호).
- `validate: d => true|"문구"` — 값 검증.
- `section` — 진행바 섹션 라벨용(인지적 단계 그룹핑).
- `gate: d => bool` — 조건부 스킵 일원화(추가기재 통째 스킵 등).
- `type:"addr"` — 구조화 주소 입력(5항).

> `renderStep`은 위 키(`store`/`type:"addr"`/`section`/`gate`/`warn`)를 분기 처리하도록 확장. 기존 `cond`(조건부 노출)와 `gate`(단계/섹션 스킵)를 혼동하지 말 것.

## 7. 검증·오류 방지
- 필수 미입력 시 **다음 차단** + 해당 필드로 포커스·스크롤 + 부드러운 문구("이 칸을 채워주세요"). `stepRequired`에 `type:"addr"` 필수 처리 포함.
- 로마자 불일치 경고를 로마자 입력 상단 배너로 강조.
- 긴급연락처=본인 번호(`phone`) 감지 시 경고.
- '잘 모르겠어요'(`unsure`)는 다음을 막지 말고 표시만 → 마지막 "직원 확인 항목"에 자동 집계.

## 8. 접근성
- 색에만 의존 금지(아이콘+텍스트 병행), 터치 타깃 48px 이상, 대비 4.5:1 이상.
- 키보드 탭 순서 = 시각 순서, 명확한 focus ring, 경고는 `aria-live`로 읽어주기.
- 하단 액션바 `[이전](좌·작게) / [다음](우·크게)` 위치 고정.
